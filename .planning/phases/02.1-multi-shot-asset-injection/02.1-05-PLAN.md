<objective>
Close the remaining Phase 02.1 gap where the project-level storyboard mood preset does not visibly propagate into downstream storyboard or multi-shot confirmation cards, even though the product model expects project config to batch-seed each segment while preserving per-card overrides.

Purpose: make project-level mood defaults function as true inherited defaults across the reachable card surfaces, while keeping each segment free to diverge.
Output: a stable inheritance chain from project config to storyboard/multi-shot cards, visible inherited values in the UI, and generation/save payloads that honor inherited mood when no local override exists.
</objective>
