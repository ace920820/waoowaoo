# Phase 02.1: Multi-Shot Asset Injection - Context

**Gathered:** 2026-04-19
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase makes the multi-shot storyboard/reference-confirmation step materially more controllable by injecting project assets into multi-shot reference-board generation. The scope is limited to character / prop / location asset binding, mood-preset participation, and card-level management behavior for each multi-shot segment during reference image generation. It does not expand into advanced omnireference inputs, downstream dialogue editing, or manual single-shot supplement flows.

</domain>

<decisions>
## Implementation Decisions

### Asset binding model
- **D-01:** For each multi-shot segment, the system should automatically preselect `场景 / 角色 / 物品` assets based on the segment script content and current segment draft metadata.
- **D-02:** Users must be able to manually adjust the preselected assets on the segment card before or after generating the reference image.
- **D-03:** The experience should feel like the traditional storyboard image-generation card model, rather than a brand-new interaction language.

### Asset priority and missing-asset strategy
- **D-04:** Asset priority order is `user manual selection > system auto-preselection > raw script parsing`.
- **D-05:** Missing assets must not block multi-shot reference generation entirely; users can continue, but the card must show a clear, visible warning state when key asset constraints are absent.
- **D-06:** The warning should live on the affected segment card itself so users can see which multi-shot segment still lacks usable asset constraints.

### Card composition and generation inputs
- **D-07:** The multi-shot reference-generation card should expose the same core control family as traditional storyboard generation: `场景 / 角色 / 物品 / 场景氛围预设 / 自定义氛围`.
- **D-08:** Asset and mood inputs belong directly in the multi-shot segment card used for reference-board generation, not buried in a separate setup page.
- **D-09:** Mood preset participation should follow the established storyboard hierarchy patterns where possible, but still be editable per multi-shot segment at the card level.

### Hover management behavior
- **D-10:** When hovering the generated image/reference preview for a multi-shot segment, users should be able to perform: `查看大图 / 重新生成 / 替换上传 / 编辑资产引用 / 移除当前引用`.
- **D-11:** Hover management should cover both image-level actions and asset-binding actions, so users do not need to leave the card to perform common corrections.
- **D-12:** Detailed management should remain lightweight and card-local first; the default interaction should stay fast enough for users reviewing many segments in sequence.

### the agent's Discretion
- Exact visual treatment of the asset chips, warning badges, and hover action tray, as long as the interaction family clearly matches the traditional storyboard card style
- Exact wording of the missing-asset warnings and helper microcopy
- Whether asset editing opens inline controls, a compact popover, or a lightweight modal, as long as the user can complete the required card-level management actions efficiently

</decisions>

<specifics>
## Specific Ideas

- The user explicitly wants the multi-shot segment card to reference the traditional storyboard image-generation card pattern, especially the control block containing `分镜氛围预设 / 自定义氛围 / 场景 / 角色`.
- The user wants asset types to include at least `人物 / 物品 / 场景`.
- The user wants hover behavior on the generated image itself to support active management, not only passive preview.
- The goal is to improve controllability and consistency for multi-shot reference-board generation without slowing the fast path back down into a heavy manual workflow.

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `ShotGroupVideoSection.tsx` already owns the multi-shot review card list, reference upload, and reference-board generation actions, making it the natural phase entry point for asset injection.
- `ShotGroupSection.tsx` already contains preview and image-management patterns for shot-group references, including hover affordances that can inform the multi-shot confirmation card.
- `StoryboardGroup.tsx` already exposes the established mood preset and custom mood interaction family that the user wants mirrored in multi-shot cards.
- `storyboard-panel-asset-utils.ts` and `ImageEditModalAssetPicker.tsx` already encode useful asset-selection patterns for character appearances and selected location images.

### Established Patterns
- Episode-level workspace data already includes `shotGroups`, so this phase can stay episode-scoped instead of introducing project-global configuration.
- Shot-group persistence already flows through `useUpdateProjectShotGroup(...)` plus `videoReferencesJson`, which is the clearest existing integration point for storing asset-binding metadata.
- The multi-shot confirmation step is already card-based and segment-based, so asset injection should extend those cards rather than introducing a separate batch editor.

### Integration Points
- The current multi-shot reference-generation action is `useGenerateProjectShotGroupImage(...)`; this phase needs to ensure the generation request has access to the selected/preselected assets and mood inputs.
- The current shot-group PATCH mutation is where per-card asset bindings and mood overrides can likely be saved before image generation.
- The current review cards already compute per-segment draft metadata; asset preselection should likely derive from that metadata plus episode/project asset libraries.

</code_context>
