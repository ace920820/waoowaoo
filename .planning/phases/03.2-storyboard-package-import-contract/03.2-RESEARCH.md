## Scope

Phase 03.2 is a contract-only implementation phase. It defines and tests the `waoo.storyboard_package` v1.0 parser, validator, and mapper used by later import API and upload UI phases.

## Existing Surfaces

- `src/lib/shot-group/template-registry.ts` defines supported templates: `grid-4`, `grid-6`, `grid-9`, and slot counts.
- `src/lib/shot-group/draft-metadata.ts` defines persisted review/video metadata fields including `referencePromptText`, `compositePromptText`, storyboard mode fields, asset bindings, mood fields, `dialogueOverrideText`, and `cinematicPlan`.
- `src/lib/shot-group/video-config.ts` defines `ShotGroupVideoMode` as `omni-reference | smart-multi-frame` and sanitizes generation options.
- `src/lib/novel-promotion/multi-shot/episode-draft-builder.ts` defines `EpisodeMultiShotDraft`, `EpisodeMultiShotDraftItem`, and `EpisodeMultiShotCinematicPlan`, which are the closest existing contract for mapping imported director segments.
- `package.json` already includes `zod`, so the contract can use Zod without adding dependencies.

## Recommended New Module Shape

Create a new package-import module under:

- `src/lib/novel-promotion/storyboard-package/schema.ts`
- `src/lib/novel-promotion/storyboard-package/parser.ts`
- `src/lib/novel-promotion/storyboard-package/validator.ts`
- `src/lib/novel-promotion/storyboard-package/mapper.ts`
- `src/lib/novel-promotion/storyboard-package/index.ts`

Tests should live under:

- `tests/unit/novel-promotion/storyboard-package/storyboard-package-parser.test.ts`
- `tests/unit/novel-promotion/storyboard-package/storyboard-package-validator.test.ts`
- `tests/unit/novel-promotion/storyboard-package/storyboard-package-mapper.test.ts`

## Contract Decisions

1. Import supports raw JSON text and Markdown fenced blocks only.
2. Markdown block info string should accept `waoo-storyboard-package+json`, and may tolerate common whitespace around the info string.
3. Do not infer arbitrary Markdown tables in Phase 03.2; return a clear missing-block error.
4. Schema top-level fields: `schema`, `schemaVersion`, `packageId`, `title`, `language`, `global`, `assets`, `scenes`.
5. `schema` must equal `waoo.storyboard_package`; `schemaVersion` must equal `1.0` for the first implementation.
6. `templateKey` must be one of `grid-4`, `grid-6`, `grid-9`.
7. Segment `targetDurationSec` defaults to 15-second production units and should be rejected if greater than 15 unless an explicit future exception field is introduced.
8. `reviewConfig` maps to the multi-shot confirmation page; `videoConfig` maps to the video production page; `cinematicPlan` carries the director shot table.
9. Asset references in Phase 03.2 remain semantic references (`externalId`, `matchName`, labels). Actual database asset matching belongs to Phase 03.3.
10. Mapper output should be API-ready but persistence-free: it can produce normalized segment payloads and draft-metadata-shaped patches, not perform Prisma writes.

## Mapper Target

The mapper should produce an internal normalized output that later API code can persist directly:

- package metadata: package id, title, language, warnings
- scenes and flattened segments
- for each segment:
  - stable import identity: `packageId`, `sceneId`, `segmentId`, `order`, `timecode`
  - shot-group fields: `title`, `templateKey`, `groupPrompt`, `videoPrompt`, `includeDialogue`, `dialogueLanguage`, `generateAudio`, `referenceMode`, `generationOptions`
  - draft metadata patch: `referencePromptText`, `compositePromptText`, storyboard mode fields, mood fields, dialogue text, `cinematicPlan`, script-derived asset references
  - shot items: `itemIndex`, `title`, `prompt`, plus optional cinematic metadata

## Validation Focus

- Valid package with one scene and one segment parses and maps.
- Raw JSON and fenced Markdown parse the same package.
- Missing fenced block in Markdown returns a typed parse error.
- Unsupported schema/version fails with actionable errors.
- Invalid `templateKey` and invalid `referenceMode` fail validation.
- More shots than template slots fails validation.
- Segment longer than 15 seconds fails validation.
- Missing required review/video/cinematic fields fails validation with a path.
- Mapper preserves prompts, storyboard mode, video config, dialogue text, cinematic plan, and shot item prompts.

## Risks

- Overfitting schema to one director sample would make future packages brittle. Keep fields explicit but allow optional descriptive fields where not required for Waoo persistence.
- Making mapper persist directly would blur Phase 03.2 and Phase 03.3. Keep mapper pure and testable.
- Free-form Markdown parsing would be fragile and hard to validate; require fenced JSON for initial import.
