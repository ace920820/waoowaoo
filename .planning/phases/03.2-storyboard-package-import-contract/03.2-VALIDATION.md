# 03.2 Validation Plan — Storyboard Package Import Contract

## Goal-Backward Validation

Phase 03.2 succeeds when later phases can rely on a stable, typed, tested package contract without touching API persistence or UI.

## Required Automated Checks

- `npx vitest run tests/unit/novel-promotion/storyboard-package/storyboard-package-parser.test.ts`
- `npx vitest run tests/unit/novel-promotion/storyboard-package/storyboard-package-validator.test.ts`
- `npx vitest run tests/unit/novel-promotion/storyboard-package/storyboard-package-mapper.test.ts`

## Required Test Cases

### Parser

- Parses raw JSON package text.
- Extracts package JSON from Markdown fenced block `waoo-storyboard-package+json`.
- Rejects Markdown without the fenced import block with a clear error.
- Rejects malformed JSON with a clear error.

### Validator

- Accepts a minimal valid v1 package.
- Rejects unsupported `schema` and `schemaVersion`.
- Rejects invalid `templateKey`.
- Rejects invalid `videoConfig.referenceMode`.
- Rejects segment `targetDurationSec > 15`.
- Rejects shot counts that exceed template slot count.
- Reports validation paths for missing required scene, segment, review, video, and shot fields.

### Mapper

- Maps review config into draft metadata fields.
- Maps video config into shot-group-style fields.
- Maps storyboard mode fields.
- Maps asset references into script-derived asset-binding references for later preview/API matching.
- Maps cinematic shots into item title/prompt payloads.
- Preserves director metadata needed for Phase 03.3 overwrite identity: `packageId`, `sceneId`, `segmentId`.

## Manual Review Checklist

- New modules are pure library code with no Prisma, no API route mutation, and no UI coupling.
- Schema names match `.planning/REQUIREMENTS.md` SBPI-01 through SBPI-12.
- Mapper output shape is documented enough for Phase 03.3 planning.
- Tests are focused and do not require network, database, browser, or worker runtime.

## Pass Condition

All listed unit tests pass and the generated contract can be consumed by Phase 03.3 without redesigning field names or parser behavior.
