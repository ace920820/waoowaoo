# 03.1-03 Summary — Cinematic Prompt Routing

Completed. Reference image, storyboard composite, and multi-shot video prompt builders now route specialized cinematic data into generation prompts.

- `tests/integration/api/specific/novel-promotion-generate-shot-group-image-assets.test.ts`

## Details

- Reference image prompt remains single-key-visual oriented and includes generated reference text, cinematic intent, visual strategy, assets, mood, style, and no-grid constraints.
- Composite storyboard prompt uses generated storyboard text, asset/mood constraints, cinematic intent, visual strategy, ordered slot prompts, and shot-level cinematic plan details.
- Video prompt prefers generated `cinematicPlan.videoPrompt` when present, treats slots as one continuous approximately 15-second beat sequence, preserves dialogue policy, and includes continuity constraints for identity, wardrobe, environment, lighting, camera direction, and blocking.
- Reference, storyboard, and video prompt builders now include a `data/镜头语言.md`-derived lens-language discipline covering concrete film grammar, 180°/eyeline continuity, spatial geography, and contradiction avoidance.
- Worker/provider behavior was left unchanged; richer prompt strings flow through existing builders and task payload paths.
