# Phase 03.1 Context: Multi-Shot Cinematic Prompting

## Problem

The multi-shot `Script->Storyboard` path already calls an LLM, but its output is centered on one shared `narrativePrompt`. That same text currently seeds the confirmation card's `辅助参考图提示词`, `剧情内容`, and initial video prompt path, which limits production quality because concept reference images, storyboard boards, and final videos need different prompt shapes.

## Locked Decisions

- Keep the existing multi-shot flow and persistence model; do not introduce a new production surface.
- Preserve backward compatibility with existing LLM outputs containing only `narrativePrompt`, `embeddedDialogue`, `shotRhythmGuidance`, and `expectedShotCount`.
- Use the existing `NovelPromotionShotGroupItem.prompt` path for shot-level prompt beats before adding new persistence tables.
- Keep review-card fields editable and preserve user overrides through rebuilds and benign refreshes.
- Scope this phase to prompt/data quality, not final rollout validation; Phase 4 remains the hardening phase.

## Desired Outcome

Each generated 15-second multi-shot segment should have:

- a concept/reference prompt for the mother image
- a storyboard prompt for the composite board
- a final video prompt for one coherent multi-shot clip
- emotional intent and visual strategy metadata
- 4-9 ordered shot beats with shot size, angle, camera movement, composition, lighting, blocking, emotional beat, and image prompt

## Acceptance Notes

- Old rows must still work.
- New rows should make `辅助参考图提示词` and `剧情内容` distinct when the model provides distinct values.
- The composite storyboard prompt and video prompt should include shot-level ordered prompts rather than only slot titles.
