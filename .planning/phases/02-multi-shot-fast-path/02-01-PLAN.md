## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workspace client -> multi-shot drafts route | Episode ID and launch timing come from untrusted client input |
| multi-shot drafts route -> Prisma shot-group writes | Draft metadata and prompt content become persisted production inputs |
| shot-group PATCH -> stored `videoReferencesJson` | Existing config can be clobbered if metadata merge is not explicit |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-02 | T | `src/app/api/novel-promotion/[projectId]/shot-groups/route.ts` | mitigate | Merge `draftMetadata` into `videoReferencesJson` explicitly so PATCH requests do not erase segment-order or placeholder status by accident. |
| T-02-03 | D | `src/lib/novel-promotion/multi-shot/episode-draft-builder.ts` | mitigate | Convert bad clip input into placeholder drafts with `sourceStatus: 'placeholder'` instead of throwing and aborting the whole episode. |
| T-02-04 | I | Stored prompt payloads in shot groups | accept | Prompts only contain already-authenticated episode script data; no new external disclosure path is introduced in this plan. |
