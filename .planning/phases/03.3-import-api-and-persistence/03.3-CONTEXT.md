# Phase 03.3 Context — Import API And Persistence

## Goal

Add backend import API for storyboard packages with preview and commit modes, writing imported segments into multi-shot shot groups and draft metadata.

## Scope

- Add import route under novel-promotion project/episode API.
- Support preview mode: parse, validate, asset-match, and show create/update plan.
- Support commit mode: create or replace imported shot groups.
- Persist `referencePromptText`, `compositePromptText`, `videoPrompt`, `dialogueOverrideText`, storyboard mode, asset bindings, `cinematicPlan`, and item prompts.
- Add integration tests for preview and commit.

## Non-Goals

- No upload UI yet; that belongs to proposed Phase 03.4.
- No package schema redesign unless Phase 03.2 discovers required contract changes.
- No silent asset creation. Missing assets should produce preview warnings or script-derived fallback.
- No automatic image/video generation after import; imported data should prefill production configuration only.

## Depends On

- Phase 03.2 — Storyboard Package Import Contract.

## Requirements

- SBPI-13 through SBPI-21 in `.planning/REQUIREMENTS.md`.
