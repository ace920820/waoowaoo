## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge insets, inline status spacing |
| sm | 8px | Compact control spacing, stacked helper text |
| md | 16px | Default field spacing, card interior rhythm |
| lg | 24px | Section padding inside production-unit cards |
| xl | 32px | Gap between multi-shot section and supplement section |
| 2xl | 48px | Major section break before manual supplements |
| 3xl | 64px | Page-level top/bottom spacing only |

Exceptions: 8px is permitted only for dense chip padding and existing token compatibility; minimum interactive target remains 44px for primary actions and add-supplement triggers.

Interaction layout rules:
- Multi-shot production units appear first as one continuous section.
- Manual single-shot supplements appear only after the full multi-shot list.
- The supplement entry card uses `lg` internal padding and `xl` top gap from the final multi-shot card so it reads as secondary.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Rules:
- Use only weights `400` and `600`.
- Keep helper copy, metadata, and status copy at 14px max.
- Prompt and dialogue editors use 16px body text to keep dense Chinese copy legible during editing.
- Section headers for “多镜头片段” and “手动补充单镜头” use Heading, not Display.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f3f4f6` | Page canvas and overall production workspace background |
| Secondary (30%) | `rgba(255, 255, 255, 0.88)` | Cards, editor wells, supplement seed card, nav-adjacent surfaces |
| Accent (10%) | `#2f7bff` | Primary CTA, active edit state, focused production unit, save actions |
| Destructive | `#cb3a3a` | Delete supplement, destructive confirmations only |

Accent reserved for: primary “继续进入视频生成” path, “保存” actions for prompt/dialogue overrides, current expanded production-unit emphasis, keyboard focus ring, and the add-supplement entry trigger. Do not use accent for passive chips, metadata tags, or every button.

Semantic rules:
- Warning states continue using the existing amber glass tone for incomplete but recoverable inputs.
- Success states continue using the existing green glass tone for saved/generated status only.
- Multi-shot and single-shot sections must share the same base surface palette; hierarchy comes from layout order and headings, not different color families.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 继续进入视频生成 |
| Empty state heading | 还没有可进入视频生成的生产单元 |
| Empty state body | 先完成多镜头片段确认，或手动补充一个单镜头分镜，再进入视频生成。 |
| Error state | 生产单元加载失败，请刷新后重试；如果仍失败，先返回上一阶段检查片段确认和参考输入。 |
| Destructive confirmation | 删除单镜头补充: 删除后不会影响多镜头片段，但该补充镜头及其视频设置会被移除。 |

Required labels:
- Supplement entry trigger on multi-shot confirmation page: `手动补充单镜头`
- Supplement entry helper copy: `只在需要额外补镜头时使用，不会切回传统整集分镜流程。`
- Single-shot supplement section heading in video stage: `手动补充单镜头`
- Single-shot supplement section body: `这些镜头作为多镜头片段之外的补充单元独立生成。`
- Video prompt field label: `视频提示词`
- Dialogue field label: `台词 / 说话内容`
- Dialogue helper copy when generated text exists: `已从剧本草稿带入，可在生成前直接改写或清空。`
- Dialogue helper copy when empty: `可选。留空时按当前生产单元的默认语音内容处理。`

Confirmation rules:
- Do not describe manual supplements as “返回分镜” or “进入传统模式”.
- Do not place dialogue editing copy on the multi-shot confirmation page.
- Handoff copy must reinforce continuity: users are continuing production, not starting a new workflow.
